# 要件定義書：Car Lovers

---

## 1. プロジェクト概要

ユーザーが自慢の愛車や街で見かけた車の写真を、詳細な車両情報（メーカー・車種・型式）とともに共有・閲覧できるプラットフォーム。プライバシーに配慮してナンバープレートをぼかしマスキングし、Cloudflare のエッジネットワークを活用した高速でセキュアな体験を提供します。

---

## 2. システム構成（技術スタック）

| レイヤー | 使用技術 | 役割 |
| :--- | :--- | :--- |
| **フロントエンド** | **Flutter** | iOS / Android / Web マルチプラットフォーム対応 |
| **API / ロジック** | **Cloudflare Workers** | サーバーレスAPI、認証ロジック、CRUD処理 |
| **データベース** | **Cloudflare D1** | SQLデータベース（投稿メタデータ、車種マスター、ユーザー情報） |
| **画像ストレージ** | **Cloudflare R2** | 車の写真（オリジナルおよびマスキング済み）の保存 |
| **画像解析** | **Workers AI** | ナンバープレートの自動検知（`detr-resnet-50` モデル） |

---

## 3. 機能要件

### 3.1 ユーザー管理・認証

* **登録:** ユーザーID（英数字・ハイフン・アンダースコア、3〜30文字）とパスワード（8文字以上）で新規登録。
* **ログイン:** ユーザーID + パスワードで認証。成功時に Workers から **JWT（JSON Web Token）** を発行。
* **パスワード保護:** PBKDF2（反復回数 100,000 回、SHA-256）でハッシュ化して D1 に保存。タイミング攻撃対策の定数時間比較を実装。
* **JWT:** HS256 署名、有効期限 24 時間。秘密鍵は `wrangler secret` で Cloudflare 上に安全に配置。
* **権限管理:** 写真の閲覧は未ログインでも可能。アップロード・投稿・削除はログインユーザーのみ。

### 3.2 投稿管理機能 (CRUD)

* **投稿 (Create):**
  * 写真選択（カメラ/ギャラリー）＋車両情報入力。
  * Workers AI でナンバープレートを自動検知し、Firebase-like マスキング処理。
  * ユーザーが手動でマスキング領域を調整可能。
* **閲覧 (Read):**
  * 一覧表示（メーカー・車種でフィルタリング可）。
  * 詳細表示（画像・車両情報・投稿日時）。
* **削除 (Delete):**
  * 投稿者本人のみ実行可能（JWT でユーザーID照合）。
  * D1 の論理削除（`deleted_at` タイムスタンプ）。
* **編集 (Update):**
  * マイページから自分の投稿のメーカー・車種名（NHTSA Autocomplete）・型式・説明を編集可能。
  * 投稿者本人のみ実行可能（JWT でユーザーID照合）。

### 3.3 画像処理・プライバシー

* **自動マスキング:**
  * Workers AI で画像内のナンバープレート座標を特定。
  * 該当箇所にセパラブルボックスブラー（radius=15）でぼかし処理し、R2 にマスキング済み画像を保存。
  * オリジナル画像も R2 に別保存（`uploads/original/` プレフィックス）。
* **手動調整:** Flutter 側でバウンディングボックスの位置・サイズを調整し、マスキング領域を再送信。

## 4. データ設計

### 4.1 データベーススキーマ（D1）

**`users` テーブル**

| カラム | 型 | 説明 |
| :--- | :--- | :--- |
| `user_id` | TEXT (PK) | ユーザーID（英数字・記号、ユニーク） |
| `password_hash` | TEXT | PBKDF2 ハッシュ（`salt:hash` 形式） |
| `created_at` | DATETIME | 登録日時 |

**`posts` テーブル**

| カラム | 型 | 説明 |
| :--- | :--- | :--- |
| `id` | INTEGER (PK) | 投稿ID |
| `user_id` | TEXT | 投稿者のユーザーID |
| `car_maker` | TEXT | 車メーカー |
| `car_model` | TEXT | 車種名 |
| `car_variant` | TEXT | 型式・グレード |
| `image_url` | TEXT | R2 のマスキング済み画像パス |
| `original_image_url` | TEXT | R2 の元画像パス |
| `description` | TEXT | 投稿コメント |
| `created_at` | DATETIME | 投稿日時 |
| `deleted_at` | DATETIME | 論理削除日時（NULL = 有効） |

**`cars_master` テーブル**

| カラム | 型 | 説明 |
| :--- | :--- | :--- |
| `id` | INTEGER (PK) | マスターID |
| `maker` | TEXT | メーカー名 |
| `model` | TEXT | 車種名 |
| `variant` | TEXT | 型式 |
| `year_from` | INTEGER | 発売開始年 |
| `year_to` | INTEGER | 発売終了年（NULL = 現行） |

### 4.2 秘匿情報の管理

* **クライアントサイド:** `--dart-define=API_BASE_URL=...` によるビルド時定数注入。デフォルト値として本番 URL をフォールバック設定。`.env` ファイル不要。
* **サーバーサイド（本番）:** `wrangler secret put JWT_SECRET` で Cloudflare Workers に安全に配置。
* **サーバーサイド（ローカル）:** `.dev.vars` ファイルを Git 管理外として使用。

---

## 5. 非機能要件

* **パフォーマンス:** Cloudflare エッジネットワークによる低遅延レスポンス。Workers AI によるサーバーサイド画像処理。
* **スケーラビリティ:** サーバーレス構成により自動スケール。D1 と R2 はサーバーレスストレージ。
* **セキュリティ:** TLS 暗号化、JWT によるエンドポイント保護、PBKDF2 パスワードハッシュ、タイミング攻撃対策。
