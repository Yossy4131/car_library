# 要件定義書：皆の車博覧会

## 1. プロジェクト概要
ユーザーが自慢の愛車や街で見かけた車の写真を、詳細な車両情報（メーカー・車種・型式）とともに共有・閲覧できるプラットフォーム。プライバシーに配慮しつつ、Cloudflare を活用した高速でセキュアな体験を提供します。

---

## 2. システム構成（技術スタック）

| レイヤー | 使用技術 | 役割 |
| :--- | :--- | :--- |
| **フロントエンド** | **Flutter** | iOS / Android / Web マルチプラットフォーム対応 |
| **API / ロジック** | **Cloudflare Workers** | サーバーレスAPI、認証ロジック、CRUD処理 |
| **データベース** | **Cloudflare D1** | SQLデータベース（投稿メタデータ、車種マスター） |
| **画像ストレージ** | **Cloudflare R2** | 車の写真（オリジナルおよび加工済み）の保存 |
| **キャッシュ/配信** | **Cloudflare Images** | 画像のリサイズ、WebP変換、配信最適化 |
| **認証補助/一時保存** | **Cloudflare KV** | 認証用OTP（ワンタイムパスワード）の一時保持 |
| **画像解析** | **Workers AI** | ナンバープレートの自動検知（AIモデル使用） |
| **メール配信** | **SendGrid** | サインイン用認証メールの送信 |

---

## 3. 機能要件

### 3.1 ユーザー管理・認証
* **サインイン方式:** パスワードレス（マジックリンクまたはOTP）を検討。
    * Workers でコードを生成し KV に保存。SendGrid 経由でメール送信。
    * Flutter 側でコード照合後、Workers から **JWT (JSON Web Token)** を発行。
* **権限管理:** 写真の閲覧は未サインインでも可能。アップロード・削除はサインインユーザーのみ。

### 3.2 投稿管理機能 (CRUD)
* **投稿 (Create):**
    * 写真選択（カメラ/ギャラリー）＋車両情報入力。
    * サーバーサイド（Workers）でナンバープレートを検知し、自動マスキング。
* **閲覧 (Read):**
    * 一覧表示、詳細表示。
    * メーカー、車種、型式による**ソート・フィルタリング機能**。
* **削除 (Delete):**
    * 投稿者本人のみが実行可能。R2 内の画像と D1 のレコードを物理/論理削除。
* **編集 (Update):**
    * 登録済みの車両情報（車種・型式等）の修正。

### 3.3 画像処理・プライバシー
* **自動マスキング:**
    * Workers AI（`detr-resnet-50` 等）を用いて画像内のナンバープレート座標を特定。
    * 特定箇所を塗りつぶし、またはボカシ処理を行った上で R2 に保存。
* **配信最適化:**
    * Cloudflare Images を利用し、デバイスに応じた最適な解像度で配信。

---

## 4. データ設計・外部連携

### 4.1 車両マスターデータ
* **外部APIの活用:** `Car Query API` 等の調査を継続。
* **独自DB (D1):** 外部に適切なAPIがない場合、オープンなデータセットから構築したメーカー・車種マスターを D1 に保持し、Flutter 側でオートコンプリート（入力補完）を実装。

### 4.2 秘匿情報の管理
* **クライアントサイド:** `flutter_dotenv` 等を使用し、APIエンドポイントやキーを Git 管理外の `.env` で管理。
* **サーバーサイド:** SendGrid APIキー等は `wrangler secret` を使用して Workers 上に安全に配置。

---

## 5. 非機能要件
* **パフォーマンス:** Cloudflare のエッジネットワークを利用し、グローバルで低遅延な画像配信を実現。
* **スケーラビリティ:** サーバーレス構成により、ユーザー増に対しても自動的にスケール。
* **セキュリティ:** 通信の TLS 暗号化、JWT による API エンドポイントの保護。

---

## 6. 開発ロードマップ（今後の検討事項）

1.  **DBスキーマ設計:** D1 における `posts` テーブルと `cars_master` テーブルのリレーション定義。
2.  **Workers AI 試作:** サーバーサイドでのナンバープレート検知精度の検証。
3.  **認証フローの実装:** SendGrid と Workers KV を組み合わせた OTP 認証のプロトタイプ作成。
4.  **Flutter UI構築:** `flutter_hooks` や `go_router` を活用した、直感的で高速な操作感の UI 実装。

### 💡 開発へのアドバイス
この構成は、インフラの運用コストを極限まで抑えつつ、AI処理までエッジで完結させる非常にモダンな構成です。

まずは **「D1 + Workers でのシンプルなテキストCRUD」** を作成し、その後 **「R2 への画像アップロード」**、最後に **「Workers AI による自動加工」** と、段階的に機能を組み込んでいくのが、デバッグもしやすくスムーズに進むはずです。
